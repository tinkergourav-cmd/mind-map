# Task management panel for ThoughtFlow (v2)

All five issues from the first review have been addressed. The reorder logic now uses a `getTaskSection()` helper that partitions tasks into the same four sections the UI renders, preventing cross-section swaps. `deleteTask` removes orphaned pins from all workspaces. Tasks store `locationWorkspaceId` alongside `locationPinId`, and `navigateToTaskLocation` passes both to `navigateToPin`. IDs use `Date.now()-${random}` to avoid collisions. `sortOrder` is assigned via `Math.max(0, ...existing) + 1`.

**Watch for:** The `T` key handler omits `SELECT` and `isContentEditable` guards that other shortcut handlers in the same file include, meaning the task panel can toggle while interacting with dropdown selects or contentEditable nodes (confirmed, minor).

**Verdict**: APPROVED

## High-level view

The five v1 concerns are resolved cleanly. The section-aware reorder, workspace-aware location tracking, pin cleanup on delete, collision-resistant IDs, and monotonic sortOrder assignment all work as expected.

One minor inconsistency remains: the `handleTaskKey` and `handleReminderKey` event handlers check only `INPUT` and `TEXTAREA` targets before firing, while other handlers in the same file (`handleTimerKey`, `handleEditModeKey`, `handlePinKey`) also check `SELECT` and `e.target.isContentEditable`. This means pressing T while focused on a `<select>` element or a contentEditable div will toggle the task panel — a cosmetic inconsistency rather than a functional bug, but the task panel itself contains `<select>` elements that could be inadvertently dismissed.

## Keyboard handler guard inconsistency

The `handleTaskKey` handler checks:

```javascript
if (e.target.tagName === 'INPUT' || e.target.tagName === 'TEXTAREA') return;
```

Other handlers in the same file (e.g., `handleTimerKey`, `handleEditModeKey`, `handleNewCardKey`) use the broader check:

```javascript
if (e.target.tagName === 'INPUT' || e.target.tagName === 'TEXTAREA' || e.target.tagName === 'SELECT' || e.target.isContentEditable) return;
```

The `handleReminderKey` handler has the same narrower pattern, so this is inherited from the existing convention for panel toggles. The task panel contains `<select>` elements for status and priority — pressing T while one is focused will close the panel and lose the user's selection context.
