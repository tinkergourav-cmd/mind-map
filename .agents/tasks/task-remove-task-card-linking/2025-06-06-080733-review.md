# Removal of task-to-card linking

The change strips out the entire task-to-card linking feature — state, logic, UI affordances, persistence, and import/export handling — while leaving the standalone task system (create, edit, delete, complete, persist, groups) intact. The approach is purely subtractive: every code path that referenced `cardTaskLinks`, `addTaskFromCard`, `locateCard`, or the related UI components is deleted. No behavioral substitution or migration path exists, which is appropriate for a feature removal.

Watch for: No remaining references to the removed symbols exist (confirmed via grep), and the build passes cleanly. The change is straightforward deletion with no new logic introduced, so the risk surface is minimal.

**Verdict**: APPROVED

## High-level view

The `cardTaskLinks` state array and all functions that operated on it (`addTaskFromCard`, `locateCard`, `getLinkedCard`) are removed from App.jsx. Every callsite — keyboard shortcut (Shift+T), context menu button, auto-save, project switching, initialization, import/export, and the card-node status-dot indicator — is cleaned up in the same pass. TaskPanel.jsx drops the three props it received for this feature (`cardTaskLinks`, `nodes`, `onLocateCard`), removes the `getLinkedCard` helper, and removes the linked-card chip UI from both list and card views. The unused `Circle` icon import is also removed.

The task CRUD lifecycle (`updateTask`, `deleteTask`, `toggleTaskStatus`, group management) is untouched except for one line removed from `deleteTask` that previously cleaned up the link. Persistence still saves `tasks` and `taskGroups` to localStorage; the `cardTaskLinks` key is simply no longer written, meaning old saved data with that key will be ignored on load (harmless).
